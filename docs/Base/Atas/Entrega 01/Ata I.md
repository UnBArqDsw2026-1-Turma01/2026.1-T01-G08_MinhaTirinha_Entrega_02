# Ata de Reunião 01 — Arquitetura de Software 

* **Data:** 25 de março de 2026.
* **Duração:** Aproximadamente 50 minutos.
* **Participantes:** Ana Carolina, Gabriel Santos, Guilherme Flyan, João Marcelo, Maria Samara, Marjorie Mitzi, Pedro Henrique e Yasmin.

## 1. Objetivo
A reunião teve como objetivo discutir e definir o tema do projeto da disciplina, além de iniciar o planejamento das atividades iniciais.

## 2. Discussões

### 2.1 Proposta inicial
* Inicialmente, foi proposta a criação de uma solução voltada ao auxílio de estudantes no desenvolvimento do Trabalho de Conclusão de Curso (TCC). 
* A ideia incluía a centralização de informações e possível uso de ferramentas de apoio, como chatbot.
* No entanto, o grupo identificou dificuldades na validação do problema, uma vez que não houve confirmação clara de que a dor é relevante para os usuários. 
* Também foi levantado que já existem recursos institucionais que atendem parcialmente essa demanda.

### 2.2 Metodologia
* Foram mencionadas metodologias como Scrum, XP e RAD. 
* Contudo, decidiu-se não definir uma abordagem neste momento, devido à falta de clareza sobre o escopo do projeto. 
* A escolha será feita posteriormente, com base em requisitos mais bem definidos.

### 2.3 Redefinição do tema
* Diante das incertezas relacionadas à proposta inicial, o grupo decidiu considerar uma nova ideia: o desenvolvimento de uma aplicação para criação de tirinhas como sugerido anteriormente pela professora.
* A mudança foi motivada pela maior viabilidade de desenvolvimento e menor dependência de validação externa.

### 2.4 Organização das atividades
* Foi definida a utilização do Trello para gerenciamento das tarefas. 
* Foram listadas atividades iniciais, incluindo: Brainstorm, Rich Picture, 5W2H, Estimativas, Storyboard, Protótipo, Testes e Glossário.
* Foi destacado que algumas dessas atividades dependem da consolidação do escopo.

### 2.5 Distribuição de tarefas
* Foi sugerida a divisão de atividades entre duplas. 
* No entanto, reconheceu-se que algumas etapas, como o brainstorm, exigem participação coletiva.

## 3. Encaminhamentos
* Consolidar a definição do tema (aplicação de tirinhas);
* Realizar novo brainstorm com foco na proposta atual;
* Levantar funcionalidades iniciais do sistema;
* Organizar e distribuir responsabilidades entre os membros;
* Agendar nova reunião para acompanhamento;
* Definir a metodologia de desenvolvimento em momento posterior.

## 4. Considerações finais
A reunião resultou na mudança do direcionamento do projeto, priorizando uma solução considerada mais viável dentro do prazo da disciplina. As próximas etapas envolvem a consolidação do escopo e o detalhamento das funcionalidades.
